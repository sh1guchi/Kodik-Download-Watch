# Изменения в форке Kodik-Download-Watch

## Новое

### Пакетная загрузка серий
Новый модуль `batch_download.py` + страница `/batch/<serv>/<id>/<data>/`.
- Выбор диапазона серий (с какой по какую) и качества (360/480/720).
- Серии качаются последовательно, сегменты каждой — пулом потоков.
- Сборка через `ffmpeg concat -c copy` (без перекодирования), результат — один ZIP.
- Живой прогресс: полоса, «плёнка» ячеек по сериям (зелёная — готово, красная — ошибка с подсказкой при наведении, пульс — качается), кнопки «Скачать ZIP» и «Отменить».
- Битая серия помечается и пропускается — задача продолжается дальше.
- Фоновые задачи с TTL 3 ч и автоочисткой временных файлов.
- Настройки в `config.py`: `BATCH_DIR`, `BATCH_JOB_TTL`, `BATCH_MAX_EPISODES`.

### Тёмная / светлая тема
- Единая дизайн-система на CSS-переменных (`static/css/app.css`), базовый шаблон `base.html`.
- Тема запоминается в `localStorage` и ставится до отрисовки — без «вспышки».
- Синхронизируется с сессией через `POST /api/theme/` (старый `/change_theme/` тоже работает).
- Все страницы переписаны: поиск, результаты (карточки с постерами), страница тайтла (мета-чипы, «Связанное», карточки озвучек), выбор серий, плеер, комната, ожидание быстрой загрузки, страница «не найдено».

## Исправленные баги оригинала

1. **`combine_segments` ронял ffmpeg** — `hwaccel='cuda'` по умолчанию падал на машинах без CUDA (и для `-c copy` не нужен). Теперь `None`.
2. **Инъекция/поломка в метаданных ffmpeg** — команда собиралась строкой через f-string, кавычки/`$()`/юникод в названии ломали её. Теперь команда — список аргументов, метаданные пишутся как текст.
3. **SSLError на длинных сериях** — на каждый HLS-сегмент создавался отдельный поток (150+ разом). Теперь `ThreadPoolExecutor` (16 воркеров).
4. **Краш быстрой загрузки из кеша** — при уже собранном файле `fast_download` возвращал `hsh` вместо кортежа `(hsh, link)`, распаковка падала. Исправлено.
5. **Нет retry/timeout при скачивании сегментов** — добавлены `raise_for_status`, timeout и 3 попытки.
6. **Path traversal в `/resources/<path>`** — путь клеился вручную, `/resources/../main.py` отдавал исходник. Теперь `send_from_directory` (→ 404).
7. **`change_seria` падал** — `int(data[0])` на строке `"start:end"`. Теперь корректный парсинг.
8. **«Смотреть вместе» был сломан** — `int(temp[0])` на `"1:12"` кидал ошибку. Теперь берётся правый край диапазона.
9. **NameError при отключённом кеше** — `ch` использовался, даже когда кеширование выключено. Теперь `ch = None` и защита во всех обращениях.
10. **Неправильный префикс кеша в быстрой загрузке** — имя бралось из `'sh'+id`, а кеш писался в `'kp'+id`. Унифицировано.
11. **Не работали хоткеи next/prev серии** — в `watch.html` `max_ser` был списком вместо числа. Исправлено, плюс хоткеи больше не срабатывают при вводе в поля.
12. **`request.referrer` без fallback** — мог быть `None`. Добавлен `or '/'`.
13. **Голые `except:`** заменены на конкретные исключения с debug-логом.

## Запуск

Ничего не изменилось:

```
pip install -r requirements.txt
python main.py
```

Для быстрой и пакетной загрузки нужен **ffmpeg** в PATH (обычная загрузка работает без него).
